## What Went Well

1. Governance behavior matured significantly across the epic:
   - explicit approve/modify gates
   - blocked progression without decision
   - canonical per-phase status model
2. Observability quality improved:
   - richer, structured run/timeline events
   - lifecycle coverage added for proposal, status, and resume flows
3. Deterministic orchestration discipline was consistently reinforced:
   - strict phase ordering
   - machine-readable errors
   - idempotency patterns for retries and duplicate actions
4. Review-loop effectiveness remained high:
   - recurring review findings were generally converted into concrete patches
   - story docs captured actionable implementation intelligence for follow-on work

## Challenges and Growth Areas

1. Story status/process drift:
   - `epic-2` is marked `done` while `2-1` and `2-3` remain `in-progress`.
   - This creates governance ambiguity and weakens reporting trust.
2. Recurring repository hygiene noise:
   - `__pycache__` / `.pyc` artifacts repeatedly surfaced in review findings.
3. Concurrency and atomicity concerns appeared repeatedly:
   - approve/advance race and stale-state transition risks were patched in places, but some items remain explicitly deferred/pre-existing.
4. Deferred technical/flow issues were tolerated across stories:
   - acceptable for MVP pacing, but they accumulate risk for Epic 3 observability work.

## Pattern Synthesis Across Epic 2 Stories

### Common Struggles

- State-transition correctness and race protection were recurring complexity points.
- Proposal revision correctness (stale vs current revision) appeared in multiple stories.
- Keeping API, CRUD state, and timeline contracts aligned required repeated hardening.

### Recurring Review Feedback

- Harden invalid/malformed persisted state handling.
- Make conflict/error contracts explicit and stable for UI behavior.
- Improve test coverage for negative paths and concurrency/idempotency behavior.
- Remove generated cache artifacts from versioned changes.

### Breakthroughs

- Canonical transition-gate model became reusable across approve/modify/block/resume.
- Resume lifecycle event instrumentation established a strong base for Epic 3 timeline transparency.
- Deterministic run-state signaling (`awaiting_user_decision`, status maps, structured events) reduced ambiguity.
